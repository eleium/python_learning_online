这三个方法都是判断字符串是否由 “数字相关字符” 组成，但识别范围从窄到宽（isdecimal() < isdigit() < isnumeric()），核心区别在于 “能识别哪些数值字符”，我用「表格对比 + 示例 + 一句话总结」讲透，新手也能快速区分：
一、核心区别对照表（最直观）
表格
方法名	    识别范围（仅返回 True 的字符）	典型示例（返回 True）	                典型示例（返回 False）
isdecimal()	仅「十进制数字」              （0-9，日常最常用的数字）	'123'、'0'	'①'、'²'、'叁'、'½'
isdigit()	十进制数字 + 「数字符号」（如①、²、³ 等）	'123'、'①'、'12³'	   '叁'、'½'、'12.3'
isnumeric()	所有「数值字符」（最广，含数字、中文数字、罗马数字、分数等）	'123'、'叁'、'Ⅹ'、'½'	'12.3'、'-45'、'1e3'
二、逐方法拆解 + 代码示例
1. isdecimal()：最严格，只认 “纯十进制数字”
核心：仅识别 0-9 这 10 个十进制数字字符，是日常判断 “纯数字字符串” 的首选；
适用场景：校验手机号、身份证号、验证码等 “仅含 0-9” 的场景。

2. isdigit()：认 “十进制数字 + 数字符号”
核心：在 isdecimal() 基础上，新增识别 “有视觉形态的数字符号”（如①、②、²、³ 等）；
注意：仍不认中文数字、罗马数字、分数。

3. isnumeric()：最宽松，认 “所有数值字符”
核心：识别所有能表示 “数值” 的字符（十进制数字、数字符号、中文数字、罗马数字、分数等）；
注意：仍不认小数点、负号、科学计数法（这些是 “符号”，不是 “数值字符”）。

三、新手易踩的坑
空字符串：三个方法对 '' 都返回 False；
非字符串调用：直接给数字（如 123.isdigit()）会报错，需先转字符串 str(123).isdigit()；
“数值”≠“数值字符”：'12.3' 是合法数值，但因含小数点（非数值字符），三个方法都返回 False。

四、实用选择建议
表格
需求场景	推荐方法
校验手机号 / 验证码（纯 0-9）	isdecimal()
识别带数字符号的场景（如①、²）	isdigit()
识别中文数字 / 罗马数字等	isnumeric()
总结
识别范围：isdecimal()（仅 0-9） < isdigit()（0-9 + 数字符号） < isnumeric()（所有数值字符）；
日常校验纯数字（如手机号）优先用 isdecimal()，最精准；
三者都不认小数点、负号等 “辅助符号”，仅认 “数值字符” 本身。
简单记：decimal 认纯数字，digit 加符号，numeric 认所有数值字符。
