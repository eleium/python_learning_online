github对新手太不友好
如果你改变里本地文件夹的路径，你完蛋了
如果你不懂配置.gitignore,你完蛋了
如果你更改了文件名，你完蛋了
你要记住bash命令；


首先进入你要上传的文件夹，然后在此打开git bash:

1.git init  #初始化这个文件夹为本地的仓库。

#如果是第一次设置相关github,必须绑定github的账号：以后就不用了
2.git config --global user.name "eleium"
 git config --global user.email "422875640@qq.com"

#把本地仓库绑定到github的对应的仓库上：有http://和ssh两种.我采用ssh的方式。
3.git remote add origin git@github.com:eleium/fish.git

#开始把本地的文件暂存，用来被github追踪。放入了购物车。可以推单独的一个或几个或全部的文件。一定要注意详细准确的文件路径和扩展名。
#具体地址在D:/python_learning/trae_teacher/.git/index
4.git add .


#把上述文件放到本地仓库，D:/python_learning/trae_teacher/.git/objects/。此时你已经把上述购物车里的文件买下，有了一个小票。
5.git commit -m"他妈的都是啥啊"


#把本地的仓库分支与github上的分支统一起来。
6.git branch -M master main

#推送本地的文件到远程的仓库里
7.git push -u origin main

#查看当前的仓库的状态
8.git status      查看当前仓库状态，
		包括已修改、已暂存、未跟踪文件。当前分支与远程分支的同步状态。
		--提交代码前检查哪些文件有变更
		--合并分支后确认冲突文件
		--任何操作前，确认仓库状态。
		
#查看远程仓库的url		
9.git remote -v  查看当前远程绑定的仓库url。

git reset 	撤销提交或取消暂存。
git clean -fd   删除所有未跟踪的文件和目录（-f,强制。-d，目录）


#如果你先建立了github上的仓库，并且有一个ReadMe.MD文件，然后再推送文件到这个仓库，就会碰到历史记录与当前不符的现象，这时候你要先拉取：
10.git pull origin main --allow-unrelated-histories 拉取并合并，把两者的历史记录统一起来，然后再次推送：
git push -u origin main



本地仓库更名、修改地址后,远程仓库没变。
1，git remote -v  查看以前绑定的仓库地址
2，git remote add origin git@github.com:eleium/远程仓库.git
如果远程仓库也变了：
git remote set-url origin git@github.com:eleium/新远程仓库.git
验证连接：
git fetch origin 测试远程连接
git branch -vv 检查分支跟踪关系。

容易犯错的情况：
fatal: remote origin already exists	重复添加同名远程仓库	先删除旧远程：git remote remove origin
Permission denied (publickey)	SSH 密钥未配置	生成并添加SSH密钥到GitHub
error: failed to push some refs	远程有本地没有的提交	先执行 git pull --rebase
本地更改丢失	误用 git reset --hard 或 git clean -fd	用 git reflog 找回历史
